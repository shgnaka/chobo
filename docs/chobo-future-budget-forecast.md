# 予算と見込み

## 1. 目的

実績管理に加えて、これから使えるお金をどう見積もるかを扱う。

## 2. 論点

### 2.1 月予算

- カテゴリ別予算の入力方法を決める
- 予算超過の判定条件を決める
- 未設定カテゴリの扱いを決める

### 2.2 支出見込み

- 引落予定を見込みに含める
- 予約済み取引の扱いを決める
- 実績との差分を見せる

### 2.3 通知と警告

- 通知条件を絞る
- 通知頻度を制御する
- ユーザーが止めやすくする

## 3. issue 分割の目安

- `Budget model`
- `End-of-month forecast`
- `Budget alert policy`
