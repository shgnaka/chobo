# 分析軸の拡張

## 1. 目的

勘定科目以外の切り口を増やして、振り返りと検索の精度を上げる。

## 2. 論点

### 2.1 タグ・目的ラベル

- 任意のタグ付与を許可する
- 予算や検索との相性を考える
- 勘定科目との二重管理を避ける

### 2.2 相手先・店舗情報

- 相手先の正規化方法を決める
- 店舗名の保持可否を決める
- 検索キーとして使えるようにする

## 3. issue 分割の目安

- `Tags`
- `Counterparty and merchant metadata`
