# 資産管理の拡張

## 1. 目的

今は扱わない資産運用や返済を、別機能として後から拡張できるようにする。

## 2. 決定事項

### 2.1 Issue #13 の子Issue状況

| Issue | タイトル | 状態 |
|-------|----------|------|
| #47 | 支払期日・月末見込み | ✅ 完了 |
| #48 | 連携の分割 | ✅ 完了 |
| #49 | 投資・ローン・分割払い | ✅ 完了（方針決定） |

### 2.2 #49 の決定

投資・ローン・分割払いは、以下の原則で既存機能を活用して扱う。

- **投資**: 通常の資産勘定として追跡
- **ローン**: 通常の負債勘定として追跡
- **分割払い**: 既存の credit_expense パターンで追跡

---

## 3. 投資の追跡

### 3.1 基本方針

投資は通常の資産勘定として追跡する。高機能なポートフォリオ管理は将来課題とする。

### 3.2 勘定設定

```
勘定種類: asset
勘定名例:
  - "資産:証券:main"
  - "asset:investment:broker"
  - "資産:暗号資産"
```

### 3.3 取引パターン

**購入**
```
種別の勘定     | 相手勘定     | 摘要
---------------+-------------+------------------
expense:investment (増加) | asset:bank (減少) | 株式購入
```

**売却**
```
種別の勘定     | 相手勘定     | 摘要
---------------+-------------+------------------
asset:bank (増加) | income:capital_gain (増加) | 株式売却益
              | asset:investment (減少) |
```

**配当・分配**
```
種別の勘定     | 相手勘定     | 摘要
---------------+-------------+------------------
asset:investment (増加) | income:dividend (増加) | 配当金
```

### 3.4 記録のヒント

- 取引の説明文にティッカーシンボルを記載（例: "AAPL 株式購入"）
- コストベースは別シートで管理
- 評価益・評価損は手動で計算

---

## 4. ローンの追跡

### 4.1 基本方針

ローンは通常の負債勘定として追跡する。自動返済スケジュールは将来課題とする。

### 4.2 勘定設定

```
勘定種類: liability
勘定名例:
  - "負債:自動車ローン"
  - "liability:loan:car"
  - "負債:住宅ローン"

任意フィールド:
  - billing_day: 請求日（月中何度かに設定）
  - payment_due_day: 支払期日
```

### 4.3 取引パターン

**借入**
```
種別の勘定       | 相手勘定     | 摘要
-----------------+-------------+------------------
asset:bank (増加) | liability:loan (増加) | ローン実行
```

**毎月返済**
```
種別の勘定       | 相手勘定     | 摘要
-----------------+-------------+------------------
liability:loan (減少) | asset:bank (減少) | ローン返済
```

**利息部分（詳細追跡する場合）**
```
利息部分を費用として記録:
  - expense:interest (増加) + liability:loan (減少)
元本部分を通常返済:
  - liability:loan (減少) + asset:bank (減少)
```

### 4.4 残高確認

月末の負債勘定残高がローンの残り元本を表す。

---

## 5. 分割払いの追跡

### 5.1 基本方針

分割払いは既存の `credit_expense` と `liability_payment` で追跡する。
自動的な残り回数の管理は将来課題とする。

### 5.2 勘定設定

```
勘定種類: liability（クレジットカード勘定）
```

### 5.3 取引パターン

**分割払いで購入**
```
種類: credit_expense
説明: "店舗名 (12回払い)"
相手勘定: expense:food または該当する費用勘定
```

**每分割払いの支払い**
```
種類: liability_payment
説明: "店铺名 分割払い 1/12"
相手勘定: asset:bank
```

### 5.4 分割払いの追跡方法

説明文に回数を含める（例: "1/12", "2/12"...）ことで手動で追跡。

## 6. 将来拡張の優先順位

### 高優先度（利用頻度高い）

1. ローンの自動返済リマインダー（payment_due_day 利用）
2. 分割払いの残り回数表示

### 中優先度（利用頻度中）

3. 投資のコストベース追跡
4. ローンの元本/利息内訳

### 低優先度（複雑）

5. ポートフォリオ全体の時価評価
6. 自動的投资記録（API 連携）
